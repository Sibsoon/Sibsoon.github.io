Modeling Mesh (5 pt.)

For the project we mostly worked with our own meshes when we were testing and developping, if we had to use something else than a sphere or a simple cube. For the modeling we used Blender 2.90 and Houdini 18.5 (Apprentice).

To validate the features, we decided to present the lighting and the cloud. Both are important pieces of the target image. To find appropriate heterogeneous models turned out to be less easy than expected. Thus we decided to do it ourself.
Similar reasons apply for the lighting.

Lightning

The Lighting for our scene is represented by an area emitter. For this particular emitter, we need a very unconventional shape. Thats why we decided to model it ourself.
The model consists of individual branches of cylindric tubes. The branches are disconnected to provide the possibility to apply different stengths and colors.
The modeling process took place in three iterations. The first iteration was a single lightning bolt. The second iteration was similar to a wooden branch. For the final and third iteration, we added lots of micro lightnings to generate the feel of electricity.

Cloud

The cloud is made in Houdini. To validate the originality of the model we will provide a run through, on how it was designed together with some screenshots during the process.
The model consist out of six modeling blocks in houdini.

pipeline


Sphere

As basis for the cloud, we took a sphere, which we subdivided for a fine grained mesh.

Sphere


Mountain

Next we applied a mountain geometry node. This modifier applies a mountain like deformation to the mesh.

Mountain


Transform

To get the model in the right scale and position we used a transformation node. Here we deformed the mesh and made it smaller in the z-axis.

Transform


Cloud

Next we applied the magic of houdini. With adding the cloud node to the pipeline. This node creates a volume representation of the polygon.

Cloud


Cloudnoise

To ge a more realistic view we additionally added the cloud noise geometry node. This node will vary the cloud based on a random seed.
Since we had some iterations for the final rendered cloud, it is likely that the preview provided in this report may differ from the final model. This is because most of the changes in the houdini project influence the seed and create new random patterns.

Cloudnoise


Volume VOP

The last node in the pipeline is used for the cloud model. It is the Volume VOP node. This node allows to change color, weights and density. We followed some tutorials to set up a realistic density distribution with the right scaling, wich is represented in the Volume VOP Node figure.
